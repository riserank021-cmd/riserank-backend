SECTION 3: EC2 DEPLOYMENT STEPS (manual — do this once)
────────────────────────────────────────────────────────────────
1. Launch EC2 t2.micro (Ubuntu 22.04 LTS) — Free Tier
2. Open ports: 22 (SSH), 80 (HTTP), 443 (HTTPS — Cloudflare)
3. SSH into instance:
   ssh -i your-key.pem ubuntu@<EC2_IP>

4. Install Node.js 18:
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs

5. Install PM2:
   sudo npm install -g pm2

6. Install Nginx:
   sudo apt-get install -y nginx

7. Clone your repo:
   sudo mkdir -p /var/www/riserank-backend
   sudo chown ubuntu:ubuntu /var/www/riserank-backend
   cd /var/www/riserank-backend
   git clone https://github.com/<your-username>/riserank-backend.git .

8. Create .env file:
   nano .env   ← copy from .env.example and fill values

9. Install production dependencies:
   npm ci --only=production

10. Seed the Super Admin (first time only):
    node scripts/seedSuperAdmin.js

11. Start with PM2:
    pm2 start ecosystem.config.js --env production
    pm2 save
    pm2 startup   ← follow the command it prints

12. Copy Nginx config:
    sudo cp nginx/nginx.conf /etc/nginx/nginx.conf
    sudo nginx -t
    sudo systemctl restart nginx

13. Set up GitHub Actions secrets (Settings → Secrets):
    EC2_HOST=<your-ec2-public-ip>
    EC2_USER=ubuntu
    EC2_SSH_KEY=<contents of your .pem file>

After deploy, verify:
    GET https://api.riserank.in/api/v1/health   → { success: true }
    GET https://api.riserank.in/api/v1/docs     → Swagger UI

SECTION 5: IMPORTANT TECHNICAL DECISIONS MADE
────────────────────────────────────────────────────────────────
- Bilingual pattern: { en: String, hi: String } embedded in each doc
  → Do NOT change this to separate collections mid-development
- Admin and User are separate collections/models intentionally
  → Mixing them causes role confusion and query complexity
- Leaderboard is snapshot-based (not real-time aggregation)
  → Real-time aggregation on every page load would kill Atlas free tier
- ActivityLog TTL = 90 days → balances analytics vs Atlas storage
- Session TTL = 180 days → needed for long-term DAU analytics
- Docker not used on EC2 → PM2 runs Node directly to save RAM
- Rate limiting is in-memory → acceptable for MVP
- OTP is hashed (bcrypt) before storing → secure against DB leaks
- FCM tokens stored on User (max 5 per user) → multi-device support
- Email failures never throw → logged but don't break auth flow
- Cron jobs run in IST (Asia/Kolkata timezone)
- S3 uploads use memory storage (multer) → no temp files on disk
- CSV import is all-or-nothing → no partial imports that leave DB dirty
- Swagger docs: persistAuthorization=true → token survives page refresh
- swagger-jsdoc version: 6.x (not 7.x) → compatible with current Express

SECTION 7: SCALING ROADMAP
────────────────────────────────────────────────────────────────
Phase 1 (0-1K DAU): Current setup
  → t2.micro, MongoDB Atlas Free, PM2 instances:1

Phase 2 (1K-10K DAU):
  → Upgrade to t3.small ($15/mo)
  → MongoDB Atlas M10 ($57/mo)
  → Add Redis Cloud (rate limiting + leaderboard cache)
  → PM2 cluster mode (instances: 'max')

Phase 3 (10K-100K DAU):
  → Add ALB + 2nd EC2 instance
  → Consider RDS for analytics (keep MongoDB for content)
  → Cloudfront CDN for S3 assets

Phase 4 (100K+ DAU):
  → Split into microservices: Auth, Content, Quiz, Analytics
  → MongoDB Atlas M30+ dedicated cluster
